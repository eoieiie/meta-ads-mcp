1. 한 줄 요약
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
하나의 광고 세트(Ad Set) 안에 있는 ACTIVE 광고 2개를 챔피언(기존)
과 챌린저(신규)로 짝지어서, 7일간 데이터를 비교해 점수를 매기는
시스템. 챌린저가 이기면 교체되고, 지면 챔피언 유지.


2. 광고 운영 방식
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

[광고 세트]
- Ad Set 1개만 사용. 예산은 Ad Set 단위로 설정.
- Ad Set 안에 여러 개의 광고(Ad)를 넣으면 Meta가 예산을 나눠 씀.

[예산]
- Ad Set 일일 예산을 Meta 광고 관리자에서 직접 설정.
- 예: 일일예산 3만원 → 2개 광고가 ACTIVE면 각각 1.5만원씩.
- 코드에서 예산을 건드리지 않음. 순수 읽기만.

[광고 ON/OFF 사이클]
- 챔피언: 항상 ACTIVE 유지.
- 챌린저: 수요일에 업로드 + 광고 OFF 상태로 시작.
  → 금요일 리포트 나온 후, 토요일에 사람이 직접 ON/OFF 결정.
  → 이기면 ON (ACTIVE), 지면 OFF (삭제 대신 일시중지).

[실제 운영 예시]
  월   화   수   목   금   토   일
  ────────────────────────────────
  챔피언 ACTIVE
  새 광고      ↑업로드
                (OFF)
                     리포트→  ↑결정
                              (이김→ON / 짐→OFF)


3. 주간 운영 사이클
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

수요일    → 새 카드뉴스 인스타에 업로드 (광고 OFF)
             피드에만 노출. 오가닉 반응 먼저 봄.

목~금요일 → 대기실. 순수 유기적 데이터 수집.

금요일    → 23:00 KST, 자동 슬랙 리포트 도착.
23:00       지난 토~금 7일간의 데스매치 결과.

토요일    → 사람이 직접 판단.
아침        "챌린저가 이겼다" → 챌린저 ON / 패자 OFF
            "챔피언 유지"     → 챌린저 OFF (다음주 새 도전자 준비)


4. 점수 계산 방식 (간단 버전)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Meta API에서 7일간의 데이터를 뽑아서 3가지 지표를 만듦.

  A) 방문단가 (CPPV) — 프로필 방문 1명 데려오는 비용
     = spendKrw ÷ 방문수
     낮을수록 좋음. "돈 대비 얼마나 효율적으로 방문을 만들었나"

  B) 저장률 (SaveRate) — 광고 본 사람 중 저장한 비율
     = 저장수 ÷ 도달수
     높을수록 좋음. "게시글이 얼마나 저장할 가치가 있나"

  C) 광고빈도 (Frequency) — 같은 사람이 본 평균 횟수
     = 노출수 ÷ 도달수
     2.0 넘으면 피로도 발생. "같은 사람에게 너무 많이 보여주고 있나"

이 지표 3개로 3개 점수를 계산:

  ① CostScore (최대 120점)
     = (챔피언CPPV ÷ 챌린저CPPV) × 60
     → 챌린저가 방문단가 효율이 얼마나 좋은지

     숫자 설명:
     - ×60: "기준점". CPPV가 동등하면 60점.
     - 최대 120점: 챌린저가 2배 효율이면 만점.
     - 즉, 60점 = 동등, 60~120 = 우세, 60 미만 = 열세.

  ② AttrScore (최대 80점)
     = (챌린저저장공유율 ÷ 챔피언저장공유율) × 40
     → 챌린저 게시글이 얼마나 더 저장+공유되고 있는지

     숫자 설명:
     - ×40: "기준점". 저장공유율이 동등하면 40점.
     - 최대 80점: 챌린저가 2배 높으면 만점.
     - 40점 = 동등, 40~80 = 우세, 40 미만 = 열세.

  ③ Penalty (차감)
     챌린저 Frequency가 2.0 넘으면 (frequency - 2.0) × 10 차감
     → 너무 많이 노출된 건 감점

     숫자 설명:
     - 기준 2.0: 같은 사람이 평균 2번 이상 보면 과노출로 판단.
     - ×10: 2.0 넘는 만큼 10점씩 차감. (3.0이면 10점, 5.0이면 30점)

  왜 60+40=100인가?
     CostScore 기준 60점 + AttrScore 기준 40점 = 100점.
     챔피언 HS가 100점 고정이므로, 두 영역 모두 "동등"해야 100점.
     하나라도 잘하면 100점 넘어서 승리. 둘 다 못하면 패배.
     즉, "100점" = 챔피언을 이기기 위한 마지노선.

  최종 점수합산:
    챌린저 HS = CostScore + AttrScore - Penalty
    챔피언 HS = 100점 (고정)

  승자 판정:
    챌린저 > 100 → 🥊 챌린저 승리 (교체!)
    챌린저 ≤ 100 → 👑 챔피언 유지 (현상유지)


5. 점수 계산 실제 예시
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

              챔피언      챌린저
  spendKrw    10,000원     5,000원
  방문수        100명       100명
  저장수         50회       100회
  노출수      15,000회    10,000회
  도달수       5,000명     5,000명

  CPPV      100원/방문     50원/방문   ← 챌린저 2배 효율
  저장률       1.0%         2.0%       ← 챌린저 2배 높음
  빈도          3.0          2.0       ← 챌린저 괜찮음

  CostScore = (100÷50)×60   = 120점
  AttrScore = (2.0÷1.0)×40  =  80점
  Penalty   = (3.0-2.0)×10  =  10점

  챌린저 HS = 120 + 80 - 10 = 190점  → 🥊 압승!
  챔피언 HS = 100점 (고정)

  → 챌린저가 방문단가 2배 효율 + 저장률 2배 높음.
    빈도가 3.0이라 10점 까였지만 그래도 여유있게 승리.


6. 결과 해석 가이드
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  챌린저 HS 100~120   → 접전. 사람이 직접 판단.
  챌린저 HS 120~150   → 챌린저가 확실히 나음.
  챌린저 HS 150+      → 압도적. 무조건 교체.
  챌린저 HS 100 미만  → 챔피언 유지. 새 도전자 준비.

  단, 빈도(Frequency)가 3.0 이상이면 실제로는 좀 더 봐야 함.
  "이미 많이 본 사람에게만 노출된 건 아닌가" 의심 필요.
